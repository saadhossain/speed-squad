Difference between localStorage and sessionStorage

Local Storage data has no expiration time. It store at your browser memory until you manually clear data.

The keys and the values stored with localStorage are always in the UTF-16 string format, which uses two bytes per character. As with objects, integer keys are automatically converted to strings.

Session Storage Data store at browser memory same as local Storage, but in the term of session storage data will be cleared every time you close the tabe or window.
Whenever a document is loaded in a particular tab in the browser, a unique page session gets created and assigned to that particular tab. That page session is valid only for that particular tab.

The difference between sessionStorage and localStorage is that localStorage data does not expire, whereas sessionStorage data is cleared when the page session ends.

Difference between Global Scope and Block Scope

Global Scope Variables declared Globally (outside any function) have Global Scope.
Global variables can be accessed from anywhere in a JavaScript program.

Variables declared with var, let and const are quite similar when declared outside a block.

Block Scope Variable declared in a function is a block scope variable. I can't access outside of the function.

ES6 introduced two important new JavaScript keywords: let and const. These two keywords provide Block Scope in JavaScript.

Variables declared with the var keyword, do not have block scope.

Describe: How Event Loop Works

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.

The browser is then set up to listen for the response from the network, and when it has something to return to you, it will schedule the callback function to be executed by inserting it into the event loop.

The Event Loop has one simple job â€” to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop. Each event is just a function callback.
